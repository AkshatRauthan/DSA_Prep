# Frequency-Based Sliding Window Pattern

## When to Use This Pattern

Use this when the problem asks for a **subarray/substring** that satisfies some **frequency constraint** on elements — e.g. "contains all chars of t", "at most k distinct elements", "all chars appear at least x times".

The key signal: you need to **match or satisfy frequencies**, not just a sum or count.

---

## The Core Idea

Instead of comparing two maps on every step (O(m) per step = O(mn) total), maintain:

```
have     => how many unique elements are currently satisfied
required => how many unique elements need to be satisfied (fixed, from constraint)
```

Window is **valid** when `have == required`.

This reduces frequency checking to **O(1) per step** — overall O(n+m).

---

## The Template

```cpp
unordered_map<char, int> need, window;

// Step 1: Build `need` from the constraint
for (char c : t) need[c]++;

int have = 0, required = need.size();
int sIdx = 0;
// track answer — adjust based on problem (minLen, maxLen, count etc.)

for (int i = 0; i < s.length(); i++) {

    // Step 2: Expand window — add s[i]
    window[s[i]]++;

    // Step 3: Check if current char just became satisfied
    if (need.count(s[i]) && window[s[i]] == need[s[i]])
        have++;

    // Step 4: While window is valid => try to shrink / record answer
    while (have == required) {

        // record answer here (min window, count, etc.)

        // Step 5: Shrink from left
        window[s[sIdx]]--;

        // Step 6: Check if removing s[sIdx] broke a requirement
        if (need.count(s[sIdx]) && window[s[sIdx]] < need[s[sIdx]])
            have--;

        sIdx++;
    }
}
```

---

## The Two Critical Checks

### On expanding (Step 3)
```cpp
if (need.count(s[i]) && window[s[i]] == need[s[i]])
    have++;
```
- `need.count(s[i])` — only care about chars that matter
- `== need[s[i]]` — only increment `have` at the **exact moment** it gets satisfied, not every time freq increases

### On shrinking (Step 6)
```cpp
if (need.count(s[sIdx]) && window[s[sIdx]] < need[s[sIdx]])
    have--;
```
- Only decrement `have` when freq **drops below** requirement — not every time it decreases

## Where to Record the Answer

Depends on the problem type:

### Minimum window
```cpp
while (have == required) {
    // record here — inside while, before shrinking
    if (i - sIdx + 1 < minLen) {
        minLen = i - sIdx + 1;
        minSIdx = sIdx;
    }
    // then shrink...
}
```

### Maximum window
```cpp
// record here — outside while, after trying to shrink
if (have == required)
    maxLen = max(maxLen, i - sIdx + 1);
```

### Count of valid windows
```cpp
while (have == required) {
    count += (n - i); // all windows ending at or after i with this sIdx are valid
    // then shrink...
}
```

## Adapting `need` for Different Constraints

The pattern stays the same — only `need` changes based on the problem.

### Match exact string (LC 76 — Min Window Substring)
```cpp
for (char c : t) need[c]++;
// required = need.size()
```

### At most k distinct elements (LC 904 — Fruit Into Baskets)
```cpp
// need is implicit — window itself is the constraint
// required = k distinct keys in window
// instead of have/required, check window.size() <= k
```

### Each char appears at least k times (LC 395)
```cpp
// required = total unique chars in s
// have++ when window[c] >= k
```

### Fixed frequency match (LC 567 — Permutation in String)
```cpp
for (char c : t) need[c]++;
// required = need.size()
// window size is fixed = t.size(), so shrink when i - sIdx + 1 > t.size()
```

---

## Common Mistakes

| Mistake | Fix |
|---|---|
| `have++` every time freq increases | Only when `window[c] == need[c]` exactly |
| `have--` every time freq decreases | Only when `window[c] < need[c]` |
| Comparing full maps each step | Use `have/required` — O(1) per step |
| Recording answer after the while loop for min window | Record **inside** the while before shrinking |
| Forgetting `need.count(c)` check | Chars not in `need` should never affect `have` |

## Problems to Practice

| Problem | Type | Twist |
|---|---|---|
| LC 76 — Minimum Window Substring | Min window | Classic template |
| LC 567 — Permutation in String | Fixed size window | Shrink when size exceeded |
| LC 438 — Find All Anagrams | Fixed size window | Collect all valid sIdx values |
| LC 904 — Fruit Into Baskets | Max window | At most 2 distinct |
| LC 395 — Longest Substring with At Least K Repeating | Max window | Each char freq >= k |
| LC 992 — Subarrays with K Different Integers | Count | atMost(k) - atMost(k-1) trick |

## Quick Reference

```
need    => what you need (built from constraint)
window  => what you have (current window frequencies)
have    => satisfied unique elements so far
required => need.size() (fixed target)

expand  => window[s[i]]++, then check if have++
shrink  => window[s[sIdx]]--, then check if have--, then sIdx++
valid   => have == required
```
